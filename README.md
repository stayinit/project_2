# Выделение бизнес-сущностей и построение статусной моделей
## Исходные данные:
Задача заключалась в анализе предметной области сервиса аутентификации и авторизации с целью выявления ключевых бизнес-сущностей. Для каждой сущности необходимо было построить статусную модель, отражающую её жизненный цикл: возможные состояния и события, вызывающие переходы между ними.
## Стек:
Draw.io, UML
## Методы и подходы:
- Анализ предметной области
- Моделирование состояний и переходов
- Использование стандартной нотации UML
## Реализация проекта:
Были выделены ключевые бизнес сущности предметной области, такие как:  
- Пользователь - представляет конечного пользователя системы. Может проходить регистрацию, авторизацию, быть в разных статусах
- Сеанс - отражает факт активного входа пользователя и хранит активый токен.
- Токен - маркер доступа, выданный пользователю после успешного входа. Имеет временной период действия, также может быть аннулирован вручную
- Роль - определяет уровни доступа пользователей системы и их динамическое изменение
- Внешний провайдер - управляет подключением и синхронизацией с внешними системами аутентификации
К этим сущностям были спроектированы статсуные модели, отражающие их возможные состояния
